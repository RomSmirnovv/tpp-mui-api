# Миграции базы данных

## Удаление поля pText (Критическая безопасность)

### Описание
Поле `pText` содержало пароли пользователей в открытом виде, что является критической уязвимостью безопасности. Это поле было полностью удалено из кода и должно быть удалено из базы данных.

### Запуск миграции

#### Способ 1: Через Node.js скрипт
```bash
cd /root/smtppapp/api
node migrations/remove_pText_field.js
```

#### Способ 2: Через MongoDB shell
```bash
mongo <your_database>
db.users.updateMany({}, { $unset: { pText: "" } })
```

#### Способ 3: Через MongoDB Compass или другую GUI
Выполните запрос:
```javascript
db.users.updateMany({}, { $unset: { pText: "" } })
```

### Что делает миграция
1. Подключается к базе данных MongoDB
2. Находит всех пользователей с полем `pText`
3. Удаляет поле `pText` из всех записей
4. Выводит статистику об удаленных полях

### Важно
- ⚠️ **Сделайте резервную копию базы данных перед запуском миграции**
- ✅ После миграции пароли будут храниться только в зашифрованном виде (bcrypt)
- ✅ Код уже обновлен и не будет создавать или использовать поле `pText`

### Проверка после миграции
```javascript
// Проверить, что поле удалено
db.users.find({ pText: { $exists: true } }).count()
// Должно вернуть 0
```
